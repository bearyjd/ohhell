# Scorekeeper Feature Design

**Date:** 2026-04-12
**Status:** Approved
**Scope:** Real-life physical game scorekeeper — single phone, no network

---

## Overview

A standalone scorekeeper mode inside the existing Flutter client. One player's phone acts as the score pad for a physical Oh Hell game. Players and rounds are configured at setup; the phone keeper enters bids at the start of each round and tricks won at the end. Running totals are calculated automatically and the final (or in-progress) scoreboard can be exported as an image + text for sharing.

---

## Scoring Rules

Standard variant (already implemented in `ohhell_engine`):

| Outcome | Points |
|---------|--------|
| Exact bid | +10 + tricks won |
| Wrong bid | tricks won only |

---

## Architecture

Self-contained feature slice inside `client/lib/features/scorekeeper/`. Zero changes to the existing online game flow.

```
client/lib/features/scorekeeper/
  providers/
    scorekeeper_provider.dart     ← ScorekeeperNotifier (Riverpod StateNotifier)
  screens/
    scorer_setup_screen.dart      ← player names + round schedule preview
    scorer_bidding_screen.dart    ← bid entry at round start
    scorer_tricks_screen.dart     ← tricks entry at round end
    scorer_leaderboard_screen.dart ← running totals + export
  widgets/
    score_table.dart              ← shared score display (screen + export card)
```

**New routes** added to GoRouter:
- `/scorer/setup`
- `/scorer/bid/:round`
- `/scorer/tricks/:round`
- `/scorer/scores`

**Home screen** gets a "Scorekeeper" button alongside the existing Join/Host options.

**New dependency:** `share_plus` (image + text sharing via native share sheet). Image capture uses Flutter's built-in `RepaintBoundary.toImage()` — no additional packages.

## State Model (`ScorekeeperNotifier`)

```dart
class ScorekeeperState {
  final List<String> playerNames;       // 3–7 players
  final List<int> roundSchedule;        // from GameConfig.defaultFor(playerCount)
  final int currentRoundIndex;
  final List<Map<String, int>> bids;    // bids[roundIndex][playerName]
  final List<Map<String, int>> tricks;  // tricks[roundIndex][playerName]
}
```

Derived values (computed, not stored):
- Per-player round scores — via `ScoreCalculator` from `ohhell_engine`
- Per-player running totals
- Winner — highest running total at game end

## Screen Flow

### Setup (`/scorer/setup`)
- Player count stepper: 3–7
- Name fields per player (defaults: "Player 1" … "Player N", all editable)
- Round schedule preview updates live: e.g. "17 rounds · 1 → 17 → 1"
- "Start Game" → navigates to `/scorer/bid/0`

### Bidding (`/scorer/bid/:round`)
- Header: round number, cards per hand (e.g. "Round 3 · 3 cards")
- One row per player: name + integer stepper (0 to cardsPerHand)
- No validation on total bids (physical play, no dealer constraint enforced here)
- Collapsed mini-table shows running totals at bottom
- "Lock Bids" → navigates to `/scorer/scores` (leaderboard, mid-round view)
- From leaderboard, "Enter Tricks" → navigates to `/scorer/tricks/:round`

### Tricks (`/scorer/tricks/:round`)
- Same layout as bidding but for tricks won
- Validation: sum of all tricks entered must equal cardsPerHand before "End Round" is enabled
- "End Round" → calculates scores, navigates to `/scorer/scores`

### Leaderboard (`/scorer/scores`)
- Full score table: players as columns, rounds as rows, running totals pinned at bottom
- Exact bids highlighted green; wrong bids neutral
- "Next Round" button (becomes "Game Over" on final round)
- "Share" button always visible (works mid-game for partial results)
- Game over state: winner highlighted with crown indicator, share button prominent

## Export

### Image
- `ScoreCard` widget wrapped in `RepaintBoundary`
- Renders full score table with dark background, player columns, round rows, totals, date, game metadata
- Captured at 2× pixel density via `RepaintBoundary.toImage()`
- Saved to temp file, shared via `share_plus`

### Text
```
🃏 Oh Hell — 17 Apr 2026
Players: Alice, Bob, Carol, Dave

       Alice  Bob  Carol  Dave
Rnd 1:   12    0    10    12
Rnd 2:    2   10    12     0
...
TOTAL:   84   62    76    48

👑 Alice wins!
Generated by Oh Hell app
```

### Share Flow
Single "Share" tap opens native share sheet with image file + text attached. Works on iOS and Android. User picks destination (WhatsApp, iMessage, etc.).

---

## Out of Scope

- Custom round schedules (auto-calculated only)
- Multiple scoring variants (standard only)
- Saving/resuming games across app restarts
- Graphics/visual polish (deferred — placeholder styling acceptable at launch)
- Dealer constraint enforcement (physical game, keeper enters actual results)

---

## Testing

- `ScorekeeperNotifier` unit tests: bid entry, tricks entry, score calculation, round progression, game end detection
- Widget tests: Setup screen validates player count bounds; Tricks screen blocks "End Round" when total ≠ cardsPerHand
- Export: text generation unit tested with known inputs/outputs
